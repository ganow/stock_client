
☆データセット仕様
key(リクエスト判別キー) -> 0~MAX_UINT_32_tの乱数
code(コード種類) -> 0x000 リスト (サーバから)
				  0x001 リクエストok (サーバから)
				  0x100  購入 (クライアントから)
				  0x101  売却(クライアントから)
				  0x400~ エラー種類：
				  404 購入額多すぎ
				  405 売却株数多すぎ
				  402 リクエスト送りすぎ
				  403 idが存在しない
				  401 key間違ってる
				  400 codeが変

id(企業id) -> 0~9
value(株価) -> MAX_UINT32_Tを超えない範囲 × 20

☆ルール
株取引ゲーム
最後に所持金が一番多い人が勝ち
所持株はカウントされないので 最後はすべて売却済みの状態に
ユーザは1ターンにつきリクエストは5回まで可能
価格リストが送られるのはターンの最初のみ
1ターン 15秒
ターンを超えたリクエストはサーバ側のkeyが一致しないので受け付けられない
所持金と所持株数と株価は自分でちゃんと把握しよう！

☆ゲーム流れ

4人が接続するとゲーム開始

サーバからuint32_t * 12行の価格リストが送られる
	key, code (0x000), [ id , value] *10

クライアントからuint32_t *4行のリクエスト
	key, code (0x100 or 0x101), id, value  (1度に1社にのみリクエスト送信)

サーバからuint32_t*12行のレスポンス(アクセプト/エラーコード)
	key, code(0x001 or 0x400~), [id, value] *10

(ターン*10)

サーバから終了宣言+結果
	key -> 0
	code -> 0x002
	rank (自分のランク)
	budget (最終所持金)
